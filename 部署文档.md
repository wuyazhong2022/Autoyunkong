# 云控系统 - 项目部署文档

## 项目概述

云控系统是一个基于 Web 的 Android 设备远程控制系统，使用 Koa + WebSocket 实现服务端，前端使用原生 JavaScript，配合 AutoJS 实现手机端自动化控制。

### 技术栈

| 组件 | 技术 |
|------|------|
| 后端框架 | Koa v2.7.0 (Node.js) |
| WebSocket | koa-websocket v5.0.1 |
| 前端 | 原生 HTML + CSS + JavaScript |
| 客户端 | AutoJS (Android自动化脚本) |
| 数据存储 | JSON文件 |

### 主要功能

- **设备管理** - 管理连接的 Android 设备，支持批量操作
- **脚本管理** - 创建、编辑、执行 AutoJS 脚本
- **任务下发** - 向设备下发快速操作命令
- **群控投屏** - 多设备屏幕实时监控和控制
- **双向消息** - Web端与设备端双向通信

## 部署指南

### 环境要求

- Node.js >= 14.x
- npm >= 6.x
- Android 手机（需要安装 AutoJS）

### 本地开发部署

```bash
# 1. 进入项目目录
cd 云控6副

# 2. 安装依赖
npm install

# 3. 启动服务器
node server.js

# 4. 访问控制台
# 浏览器打开 http://127.0.0.1:3007
```

### 服务器部署（Linux）

#### 方式一：直接运行

```bash
# 1. 上传项目到服务器
scp -r ./云控6副 user@your-server:/home/user/

# 2. SSH 登录服务器
ssh user@your-server

# 3. 进入项目目录
cd /home/user/云控6副

# 4. 安装依赖
npm install

# 5. 后台运行服务
nohup node server.js > server.log 2>&1 &

# 6. 检查服务状态
ps aux | grep server.js
tail -f server.log
```

#### 方式二：使用 PM2（推荐）

PM2 是一个 Node.js 进程管理器，支持负载均衡、自动重启等功能。

```bash
# 1. 全局安装 PM2
npm install -g pm2

# 2. 启动服务
pm2 start server.js --name koaws

# 3. 设置开机自启
pm2 save
pm2 startup

# 4. PM2 常用命令
pm2 list              # 查看进程列表
pm2 logs koaws        # 查看日志
pm2 restart koaws     # 重启服务
pm2 stop koaws        # 停止服务
pm2 delete koaws      # 删除进程
```
pm2 start server.js --name autojs-server
pm2 stop autojs-server
pm2 delete autojs-server
#### 方式三：使用 Docker（可选）

如果需要使用 Docker 部署，创建 `Dockerfile`：

```dockerfile
FROM node:18-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install

COPY . .

EXPOSE 3007

CMD ["node", "server.js"]
```

Docker 部署命令：

```bash
# 构建镜像
docker build -t koaws .

# 运行容器
docker run -d -p 3007:3007 --name koaws koaws
```

## 配置说明

### 服务器配置

服务器配置在 `server.js` 中，默认配置：

```javascript
const PORT = 3007;           // 服务端口
const HOST = '0.0.0.0';      // 监听地址
```

### WebSocket 连接

客户端（AutoJS）需要配置服务器地址：

```javascript
// 在 autojsWS.js 中修改
const WS_URL = 'ws://你的服务器IP:3007';
```

## AutoJS 客户端配置

1. 在 Android 手机上安装 AutoJS
2. 复制 `autojsWS.js` 到手机
3. 修改脚本中的服务器地址：
   ```javascript
   const WS_URL = 'ws://你的服务器IP:3007';
   ```
4. 在 AutoJS 中打开脚本并运行
5. 首次运行需要在手机上授权必要的权限

## 防火墙配置

确保服务器开放 3007 端口：

```bash
# Ubuntu/Debian
sudo ufw allow 3007

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=3007/tcp
sudo firewall-cmd --reload
```

## 常见问题

### 1. 设备连接不上

- 检查服务器是否正常运行：`pm2 logs koaws`
- 检查防火墙是否开放 3007 端口
- 检查客户端配置的 IP 地址是否正确
- 检查手机和服务器网络是否互通

### 2. WebSocket 连接失败

- 确保客户端和服务端的 WebSocket 版本兼容
- 检查是否被防火墙或代理阻断

### 3. 脚本无法执行

- 检查 AutoJS 是否有足够的权限
- 检查脚本语法是否正确
- 查看服务器日志获取详细错误信息
